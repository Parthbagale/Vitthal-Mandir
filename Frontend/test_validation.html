Registration Validation Test

Test Cases:

✅ Valid Test Data:

  • Name: John Doe (letters only)
  • Username: john_doe123 (alphanumeric + underscore)
  • Email: john@gmail.com (letters in both parts)
  • Mobile: 9876543210 (10 digits, starts with 6-9)
  • Aadhaar: 234123412346 (12 digits, valid checksum)

❌ Invalid Test Data (should be rejected):

  • Name: John123 (contains numbers)
  • Email: 123@456.com (no letters in local/domain)
  • Mobile: 1234567890 (doesn't start with 6-9)
  • Aadhaar: 123456789012 (invalid checksum)

📋 How to Test:

  1. Open index.html in your browser
  2. Click the "Register" button in the navbar
  3. Try entering invalid data (see examples above)
  4. Check browser console (F12) for validation logs
  5. Verify that form submission is blocked with error messages
  6. Try valid data to confirm it passes validation

🔍 What to Look For:

  • ✓ Red border and background on invalid fields
  • ✓ Error messages below each field
  • ✓ Console logs showing validation details
  • ✓ Form submission blocked when validation fails
  • ✓ Scroll to first error field
  • ✓ Real-time validation on blur (when leaving field)
